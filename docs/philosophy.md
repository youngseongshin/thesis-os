# Philosophy

Thesis OS starts from one claim:

> Good investment judgment should be explicit, evidence-backed, auditable, and improvable.

The system is not designed to replace human judgment. It is designed to make judgment visible enough that it can be inspected, challenged, measured, and improved.

## Principles

### 1. Evidence Before Opinion

The system separates facts, assumptions, analysis, and actions. A thesis should not be accepted merely because it sounds plausible. It needs evidence records with source, timestamp, confidence, and interpretation boundaries.

### 2. Thesis As A Living Object

A thesis is not a static memo. It has assumptions, supporting evidence, risks, invalidation conditions, actions, and update history. A thesis can strengthen, weaken, drift, or be retired.

### 3. Predictions Must Be Registered Before Outcomes

If a system explains everything after the fact, it has no memory discipline. Thesis OS records predictions before the market outcome is known.

### 4. Feedback Is Part Of Research

Research is incomplete until the system checks what happened. The feedback loop evaluates direction, timing, drawdown, maximum favorable excursion, relative performance, and failure mode.

### 5. Agents Need Clear Boundaries

Alpha collects evidence. Gyeokja makes investment judgments. Arki maintains the operating system. Each agent has a role, skill scope, and output contract.

### 6. Private Data Stays Private

Open-source code should expose methods, schemas, and examples, not real accounts, chat logs, private companies, credentials, cookies, or paid raw data.

## Non-Goals

- Not an autonomous trading bot
- Not a promise of alpha
- Not financial advice
- Not a replacement for licensed professionals
- Not a place to store secrets
